 # O que foi aprendido no Módulo 02

Introdução

Nesse módulo eu comecei a perceber que o ritmo da Alpha é realmente acelerado. Não era só digitar comandos e seguir a aula: eu precisava entender o que estava acontecendo por trás de cada coisa. Foi o módulo em que mais testei, criei pastas, apaguei pastas sem querer, me perdi, comecei do zero e fui entendendo que aprender programação envolve um pouco de caos no início. No fim, fez sentido.


Aula 01 – Primeiros comandos no Terminal

No começo dessa aula eu estava completamente perdido, tentando entender o que estava acontecendo e ainda torcendo para ninguém me chamar para explicar nada. Aos poucos fui entendendo a lógica do terminal. Foi aqui também que decidi instalar Linux e testar tudo de verdade. Acabei gostando da ideia de usar terminal porque tudo acontece mais rápido, mesmo sem interface visual.

Comandos que mais usei:

pwd    → mostra onde eu estou
ls     → lista arquivos da pasta atual
cd     → entra em uma pasta
mkdir  → cria uma pasta nova
touch  → cria um arquivo vazio

Aula 02 – Manipulação de arquivos pelo terminal

Nessa aula tive que criar, mover e apagar arquivos apenas pelo terminal. Foi prático, mas exigiu decorar bem os comandos. Mesmo tentando, ainda não consegui deixar tudo organizado, mas já deu para entender o funcionamento melhor.

Comandos importantes

echo "texto" > arquivo     → cria arquivo com texto
mv arquivo destino         → move ou renomeia
rm arquivo                 → remove arquivos
rmdir pasta                → remove pastas vazias


Aula 03 – Introdução ao Git

Essa foi a aula em que eu percebi que o Git não era tão simples quanto parecia. Eu tive que repetir bastante os comandos até entender o que cada um fazia. Não entendi tudo, mas já consigo me virar bem.

Comandos usados

git init                  → transforma a pasta em repositório
git status                → mostra mudanças
git add .                 → prepara os arquivos
git commit -m "msg"       → cria nova versão
git log                   → mostra histórico

Aula 04 – GitHub e GitHub Pages

Essa aula fez tudo parecer mais real. Subir o repositório para o GitHub deu aquela sensação de estar fazendo algo profissional. Apanhei um pouco para conectar o repositório local, mas quando funcionou, pareceu lógico. Também criamos um site usando GitHub Pages, o que foi estranho e legal ao mesmo tempo.

Comandos e passos

git remote add origin url   → conecta ao GitHub
git push -u origin main     → envia arquivos ao repositório remoto

Aula 05 – Prática intensa no terminal

Essa aula foi basicamente para reforçar tudo o que já tínhamos aprendido. Testei bastante, travei no meio do caminho, mas quanto mais repetia, mais automático ficava. Deu pra perceber que terminal é prática, não teoria.

Aula 06 – Organização e boas práticas

Aqui eu percebi que talvez parte da minha bagunça seja culpa minha. Aprendi sobre estrutura de pastas, nomeação e organização básica. Ainda tropeço às vezes, mas deu para entender como evitar confusão quando o projeto cresce.

Aula 07 – Branches e colaboração

Essa aula foi como um teste da vida real: branches, revisões, pull requests e cuidado para não quebrar o projeto da equipe. Criar branches foi tranquilo; o complicado é lembrar de mudar de branch antes de editar algo. Mesmo assim, foi uma das aulas que mais fez sentido no contexto profissional.
